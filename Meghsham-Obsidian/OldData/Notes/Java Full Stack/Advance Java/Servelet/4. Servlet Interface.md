The **Servlet Interface** in **Advanced Java** (Java EE/Jakarta EE) is the core interface for handling HTTP requests and responses in Java web applications. It is part of the **javax.servlet** package and defines the basic structure for servlets.

## **Methods of Servlet Interface**

The `Servlet` interface has five key methods:

1. **`void init(ServletConfig config) throws ServletException`**
    
    - Called once when the servlet is first initialized.
    - Used for one-time setup like loading configuration settings.
    - Example:
        
        ```java
        public void init(ServletConfig config) throws ServletException {
            System.out.println("Servlet Initialized");
        }
        ```
        
2. **`void service(ServletRequest req, ServletResponse res) throws ServletException, IOException`**
    
    - Called for each request to handle client-server communication.
    - Processes both `GET` and `POST` requests unless overridden by `HttpServlet`.
    - Example:
        
        ```java
        public void service(ServletRequest req, ServletResponse res) throws ServletException, IOException {
            res.getWriter().println("Hello, Servlet!");
        }
        ```
        
3. **`void destroy()`**
    
    - Called when the servlet is being removed from memory (e.g., server shutdown).
    - Used for resource cleanup like closing database connections.
    - Example:
        
        ```java
        public void destroy() {
            System.out.println("Servlet Destroyed");
        }
        ```
        
4. **`ServletConfig getServletConfig()`**
    
    - Returns the `ServletConfig` object containing servlet configuration details.
    - Example:
        
        ```java
        public ServletConfig getServletConfig() {
            return this.config;
        }
        ```
        
5. **`String getServletInfo()`**
    
    - Returns information about the servlet (like version or author details).
    - Example:
        
        ```java
        public String getServletInfo() {
            return "Basic Servlet by Developer";
        }
        ```
        

## **Example of Servlet Implementation**

Since the `Servlet` interface must be implemented manually, we often use **GenericServlet** or **HttpServlet** instead. Hereâ€™s an example using `HttpServlet`:

```java
import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

public class MyServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        resp.getWriter().println("Welcome to Advanced Java Servlet!");
    }
}
```

### **Conclusion**

The `Servlet` interface provides the foundation for Java web applications, defining lifecycle methods (`init`, `service`, `destroy`) and configuration methods (`getServletConfig`, `getServletInfo`). However, most developers extend `HttpServlet` for better HTTP handling.
